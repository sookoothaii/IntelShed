# Deployment

WorldBase runs on Windows (primary dev), Linux, and Docker. Pick the path that fits your platform.

---

## Windows (primary dev)

```powershell
.\start.ps1
```

This kills stale processes, starts the backend (uvicorn + reload) and frontend (Vite), waits for `/api/health/ping`, then runs the smoke test.

- UI: http://localhost:5176
- API: http://localhost:8002/docs
- Smoke test: `.\scripts\smoke-test.ps1`

### First-time setup

1. Copy `backend/.env.example` → `backend/.env` (fill in API keys).
2. Copy `frontend/.env.example` → `frontend/.env` (Cesium Ion token required).
3. Create venv: `python -m venv backend\venv` then `backend\venv\Scripts\pip install -r backend\requirements.txt`.
4. Install frontend deps: `cd frontend && npm install`.

---

## Linux / macOS

### First-time setup

```bash
# 1. Python venv + backend deps
python3 -m venv backend/venv
backend/venv/bin/pip install -r backend/requirements.txt -r backend/requirements-dev.txt

# 2. Frontend deps
cd frontend && npm install && cd ..

# 3. Env files
cp backend/.env.example backend/.env      # fill in API keys
cp frontend/.env.example frontend/.env    # Cesium Ion token

# 4. Pre-commit hooks (optional but recommended)
./scripts/install-hooks.sh
```

### Start

```bash
./scripts/start.sh
```

This is the POSIX equivalent of `start.ps1`:
- Kills stale uvicorn/vite processes
- Starts backend with reload (same excludes as Windows)
- Waits for `/api/health/ping`
- Starts frontend on port 5176
- Opens browser if `xdg-open` or `open` is available

Skip the smoke test with `--skip-smoke`.

### Stop

```bash
kill $(pgrep -f "uvicorn main:app") $(pgrep -f "vite.*5176")
```

## Docker (optional)

Docker is an alternative for containerized deployments. The Windows PowerShell path remains the primary dev workflow.

### Quick start

```powershell
.\scripts\start-docker.ps1
```

Or manually:

```bash
docker compose up -d --build
```

- Browser: https://localhost (accept the internal-CA cert once)
- Pi sync: https://<pc-lan-ip>/api/node/ingest

### What runs

| Service | Image | Notes |
|---------|-------|-------|
| backend | `worldbase-backend:local` | FastAPI on :8002, non-root user, healthcheck |
| web | `worldbase-web:local` | Caddy serves the Vite SPA + reverse-proxies /api |
| redis | `redis:7-alpine` | Rate-limit storage (local only) |

### Volumes

- `worldbase-db` — persistent SQLite at `/data/worldbase.db`
- `caddy-data` / `caddy-config` — TLS cert storage

### Env

Secrets come from `backend/.env` (via `env_file`) and the repo-root `.env` (auto-generated by `start-docker.ps1`). Never bake secrets into the image.

See `.env.docker.example` for the compose-level variables.

## CI (GitHub Actions)

The CI pipeline (`.github/workflows/ci.yml`) runs on every push/PR to `main`:

| Job | What | Platform |
|-----|------|----------|
| frontend | Build (tsc + vite) | Ubuntu |
| frontend-quality | ESLint + Prettier | Ubuntu |
| frontend-test | Vitest + RTL | Ubuntu |
| frontend-e2e | Playwright | Ubuntu |
| backend | Import + compile | Ubuntu |
| backend-quality | Ruff + mypy | Ubuntu |
| backend-tests | Full pytest suite | Ubuntu |
| pre-commit | ruff + tsc hooks | Ubuntu |

`backend-tests` runs the full pytest suite (`python -m pytest -v --tb=short --maxfail=50 -q`) with `continue-on-error: true` initially to avoid blocking while optional-dependency tests are stabilised.
